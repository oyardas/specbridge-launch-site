# Pilot Offer

Status: Draft
Purpose: Early customer pilot packaging

## Pilot name

SpecBridge AI Controlled Tender Intelligence Pilot

## Pilot objective

Validate whether SpecBridge AI can reduce tender review effort, improve risk visibility and produce useful executive, technical and sales/presales decision-support outputs.

## Pilot duration

Suggested options:

- 30 days: lightweight validation
- 60 days: deeper workflow validation
- 90 days: multi-team evaluation

## Pilot input

Pilot should use:

- Synthetic demo RFPs
- Anonymized real tender examples
- Redacted customer documents
- Non-confidential historical tender samples

Real confidential tenders should not be processed unless legal/KVKK/security terms are approved.

## Pilot outputs

- Executive summary
- Bid/no-bid signal
- Technical fit analysis
- Compliance matrix
- Gap list
- Vendor bias findings
- Clarification questions
- Commercial/legal/delivery risk notes
- Action plan
- Export package review

## Success criteria

Pilot should answer:

1. Does the output save review time?
2. Are the identified risks useful?
3. Are clarification questions actionable?
4. Does the executive report help management decision-making?
5. Is the technical report useful for presales?
6. Is the sales/presales report useful for bid strategy?
7. What must be improved before production use?

## Pilot boundary

The pilot is not a production SaaS deployment.

The pilot does not provide legal advice, final compliance approval or guaranteed tender success.

## Follow-up decision

At the end of the pilot, classify result as:

- Continue to paid pilot
- Convert to production design partner
- Extend technical validation
- Pause / no fit
