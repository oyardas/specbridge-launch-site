# R-LAUNCH.7 Professional Site Roadmap

Status: planning note
Scope: post-multilingual professionalization layer

## Positioning

SpecBridge AI is currently a controlled demo and pilot-preparation product, not a production SaaS platform.

The public site should prioritize trust, clear product value, controlled pilot conversion, sample output evidence, and legal/data-handling clarity.

It should not prematurely present self-service login, public upload, or production SaaS claims.

## Recommended sequence

### R-LAUNCH.7-A Brand Identity Foundation

Add the basic professional identity layer:

- Approved logo placement.
- Favicon and app icons.
- Open Graph and social preview image.
- Color tokens.
- Typography refinement.
- Button and card polish.
- Footer brand consistency.

This should be the next visual milestone because it improves credibility without changing backend, upload, contact, or authentication behavior.

### R-LAUNCH.7-B Product Visual Surface

Add concrete product evidence:

- Sample executive report preview.
- Sample technical report preview.
- Bid / No-Bid decision card screenshot.
- Risk heatmap screenshot.
- Clause compliance matrix screenshot.
- Controlled demo package preview.

This should come before video, because screenshots define what the video should explain.

### R-LAUNCH.7-C Explainer Video Readiness

Prepare a short video layer after brand and product visuals stabilize.

Recommended format:

- 45 to 60 seconds.
- Problem: tender and RFP reviews are slow, inconsistent, and risky.
- Solution: SpecBridge AI turns documents into structured decision packages.
- Value: faster bid/no-bid decision, clearer risk view, better clarification questions, and better presales alignment.
- Trust: controlled pilot, no public confidential upload, privacy and KVKK readiness.
- CTA: request controlled pilot discussion.

The initial site section can use a static video placeholder, poster image, and script-ready copy. Actual video production should follow storyboard approval.

### R-LAUNCH.8 Pilot Conversion Layer

Add conversion components:

- Request Pilot Access CTA.
- Pilot scope summary.
- Demo meeting request flow.
- FAQ.
- Pricing or pilot package hypothesis.
- Target persona routing.
- Security and data-handling summary.

### Later: Authentication / Login

Real user login should not be added during the static landing phase.

Preconditions:

- Backend scope approved.
- Authentication boundary designed.
- KVKK and data-processing path finalized.
- File upload and retention policy implemented.
- Security baseline defined.
- Pilot customer workflow approved.

Until then, use Request Pilot Access rather than Login.

## Additional professional headings to consider

- Product overview.
- Use cases by role.
- Sample report library.
- Security and privacy model.
- Data retention and deletion policy.
- FAQ.
- Pilot process.
- Pricing hypothesis.
- Controlled availability.
- Integrations and export formats.
- Accessibility and performance.
- SEO and social metadata.
- Analytics and conversion tracking.
- Contact and demo request flow.
- Legal document hub.
- Changelog and launch notes.

## Guardrails

Do not add public file upload, real login, production SaaS claims, customer logos, testimonials, or real tender/customer data before the underlying operational and legal readiness is complete.
