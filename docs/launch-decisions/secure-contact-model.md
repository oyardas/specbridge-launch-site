# Secure Contact Model Before Backend Forms

Status: Draft
Scope: Public launch website

## Principle

Do not open a backend contact form until data processing, retention, consent, logging, spam protection, and breach response decisions are finalized.

## Launch-safe model

For first launch, use a low-data-contact model:

1. Public email alias
2. Mailto CTA
3. Optional calendar booking link only after privacy review
4. No file upload
5. No free-text tender upload box
6. No live AI analysis endpoint
7. No CRM auto-ingestion until retention model is approved

## Recommended contact CTA

Primary CTA:

Request a private demo

Contact text:

For demo requests, pilot discussions, or sample report access, contact us by email. Please do not send confidential tender documents, personal data, commercial secrets, or production files before a pilot agreement and data handling model are confirmed.

## Email alias options

Preferred:

- hello@specbridge.ai
- demo@specbridge.ai
- pilot@specbridge.ai

Recommended use:

- hello@ for general website
- demo@ for demo booking
- pilot@ for controlled pilot conversations

## Minimum email controls

Before publishing the alias:

1. Domain SPF enabled
2. DKIM enabled
3. DMARC enabled
4. MFA on mailbox/admin account
5. No shared mailbox password
6. Auto-forwarding disabled or controlled
7. Sensitive file warning in auto-reply
8. Retention rule documented

## Auto-reply draft

Thank you for contacting SpecBridge AI.

For security and confidentiality reasons, please do not send confidential tender documents, personal data, commercial secrets, or production files by email before a pilot agreement and data handling process are confirmed.

We will review your message and respond with the appropriate next step.

## Contact form decision gate

A website contact form can be considered only after:

1. KVKK/privacy text is finalized
2. Data controller/processor role is clarified
3. Submission storage location is known
4. Retention period is defined
5. Deletion request process is defined
6. Spam and abuse protection is selected
7. Breach response process is documented
8. Export and audit process is documented
